**لا تبدأ باسم التوزيعة.** ابدأ بثلاثة أسئلة أساسية:  
1. **دورة الحياة (Lifecycle):** كم سنة تريد البقاء على نفس الإصدار؟ (1-2 سنة أم 5-10 سنوات؟)  
2. **قدرة الفريق:** ماذا يستطيع فريقك إدارة فعلياً؟ (مسؤول واحد أم فريق DevOps؟)  
3. **قيود الواقع:** كهرباء غير مستقرة، إنترنت ضعيف، عتاد محدود؟  

**قاعدة ذهبية:** اجعل **80-90% من الخوادم على توزيعة أساسية واحدة**، مع استثناءات مبررة فقط (تطبيقات بائعين، عقد Kubernetes).  

**في الدول النامية:** أعطال الكهرباء، الأقراص، ECC، والتهوية تُسقط الشركات أكثر من **apt vs dnf**.  

**أي توزيعة ستفشل** بدون **سياسة تشغيل** تشمل: تحديثات مُدارة، نسخ احتياطي مُختبَر، مراقبة، توثيق، أتمتة.

***

## 1) المبادئ الأساسية 

### 1.1 **وحّد قدر الإمكان**  
توزيعة أساسية لـ 80-90% من الخوادم. استثناءات فقط لـ:  
- برمجيات تفرض توزيعة (ERP، cPanel).  
- Immutable OS لعقد Kubernetes.  

### 1.2 **Lifecycle قبل الحزم**  
Debian: ~5 سنوات | Ubuntu LTS: 5-10 سنوات | RHEL-like: 10 سنوات.

### 1.3 **السياسة تغلب الأدوات**  
التوزيعة تقلل أخطاء البشر، لكنها لا تعوّض:  
- تحديثات أمنية منتظمة  
- Backup مُختبَر  
- Monitoring + Runbooks  
- Ansible أساسيات  

### 1.4 **الفيزياء أهم من التوزيعة**  
UPS + ECC + RAID + OOB = أولوية في الدول النامية.

***

## 2) إطار قرار: 8 أسئلة تشغيلية

1. **مدة البقاء على الإصدار؟** (1-2 أم 5-10 سنوات)  
2. **حجم الفريق؟** (1 شخص أم DevOps؟)  
3. **SELinux مطلوب؟** (نعم → RHEL-like | لا → Debian/Ubuntu)  
4. **Mutable أم Immutable؟**  
5. **إنترنت ضعيف؟** (Mirror محلي + تحديثات دفعات)  
6. **كهرباء غير مستقرة؟** (UPS Online + shutdown تلقائي)  
7. **بائع يفرض توزيعة؟**  
8. **خطة الترقية؟** (Rebuild أم In-place؟)

***

## 3) نظام **Tiers** التشغيلي

| **Tier** | **خوادم** | **الفريق** | **الخصائص** | **المتطلب الأدنى** |
|----------|------------|-------------|--------------|---------------------|
| **A** | 1-5 | جزئي | يدوي | Backup + تحديثات شهرية |
| **B** | 5-15 | 1-2 | أتمتة بسيطة | Ansible + Monitoring |
| **C** | 15-30 | 2-3 | فصل خدمات | CI + سجلات مركزية |
| **D** | 30-100 | 3-5+ | SRE كامل | IaC + Immutable |

**💡 واقع الدول النامية:** معظم الشركات في **Tier A-B** (1-15 خادم).

***

## 4) قاموس مصطلحات أساسي

| **المصطلح** | **التعريف** | **مثال** |
|--------------|--------------|----------|
| **LTS** | دعم طويل الأمد | Ubuntu 24.04 (5-10 سنوات) |
| **Immutable OS** | نظام قراءة فقط + snapshots | MicroOS |
| **SELinux** | تحكم وصول إلزامي | Rocky (Enforcing افتراضي) |
| **Transactional Updates** | تحديثات ذرية + rollback | MicroOS |
| **ABI** | توافق ثنائي تطبيقات | RHEL-like |

## 5) العتاد - أهم من التوزيعة في الدول النامية

### 5.1 **الكهرباء **
```
UPS Online (Double Conversion) + 15-30 دقيقة بطارية
إغلاق تلقائي: NUT/apcupsd
مراقبة: انقطاعات + حرارة UPS
```

### 5.2 **RAM: ECC للقواعد بيانات**
```
PostgreSQL/MySQL/Redis → ECC إجباري
Web عادي → مستحسن
```

### 5.3 **التخزين**
```
NVMe → Software RAID (mdadm/ZFS)
SATA HDD → mdadm أو Hardware RAID
RAID ليس Backup!
```

### 5.4 **OOB Management (إدارة خارج النطاق)**
```
Dell iDRAC | HPE iLO | Supermicro IPMI
🔒 VLAN منفصل + VPN فقط
```

### 5.5 **متطلبات حسب Tier**
| **Tier** | **UPS** | **ECC** | **RAID** | **OOB** |
|----------|---------|---------|----------|---------|
| A | Line-Interactive | مستحسن | RAID 1 | مستحسن |
| B | Online | نعم | RAID 1/10 | نعم |
| C-D | Redundant | نعم | RAID 10 + Hot Spare | نعم + Automation |

## 7) توصيات حسب **Tier** + سيناريوهات

### **Tier A (1-5 خوادم): شركة صغيرة جدًا**
```
✅ Ubuntu 24.04 LTS (فريق مبتدئ)
✅ Debian 13 (استقرار مطلق)
⚠️ Debian 12 (عتاد 32-bit قديم)
```

### **Tier B (5-15 خادم): SaaS صغير**
```
توزيع: LB×2 | Web×4 | DB×2 | Backup×1
✅ Debian 13 (الأفضل)
✅ Ubuntu 24.04 LTS (سهولة)
```
**الفيزياء:** UPS Online + ECC للـ DB + Software RAID.

### **Tier C (15-30 خادم): استضافة محلية**
```
توزيع: KVM×6 | Storage×3 | Web×8 | Mail×4
✅ Rocky Linux (cPanel/Plesk + SELinux)
✅ AlmaLinux (بديل ممتاز)
```
**الفيزياء:** RAID 10 + Hot Spare + OOB.

### **Tier C→D: Kubernetes (20+ خادم)**
```
✅ MicroOS (Transactional updates)
✅ Fedora CoreOS (Auto-updating)
```
**شرط:** IaC + أتمتة قوية.

## 9) متى تخلط التوزيعات؟

### ✅ **خلط مفيد**
```
Production → Debian/Rocky
CI/CD → Fedora Server
Kubernetes Workers → MicroOS
```

### ❌ **خلط ضار**
```
فريق 1-2 شخص بدون أتمتة
خدمات متشابهة بتوزيعات مختلفة
```

**القاعدة:** كلما صغر الفريق، كلما زاد التوحيد.

## الخلاصة التنفيذية النهائية

```
التوزيعة المناسبة + عتاد سليم + سياسة تشغيل = نجاح
التوزيعة وحدها ≠ نجاح
```

**للشركات الصغيرة (Tier A-B):**  
**Ubuntu LTS** (سهولة) | **Debian 13** (استقرار)  

**للمتوسطة (Tier C-D):**  
**Rocky/AlmaLinux** (توحيد + SELinux) | **MicroOS** (Kubernetes)
