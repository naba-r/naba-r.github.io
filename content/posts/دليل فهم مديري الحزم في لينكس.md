## من حفظ الأوامر إلى فهم المنطق

مديرو الحزم في لينكس لا ينبغي التعامل معهم كقائمة أوامر للحفظ:

`apt install`
`pacman -S`
`dnf upgrade`

فهذا يخلق معرفة هشة تنهار عند أول اختلاف بين التوزيعات.

الفهم الحقيقي يبدأ عندما تدرك أن هذه الأدوات ليست أوامر متفرقة،
بل أنظمة تفكير مختلفة تؤدي الوظيفة نفسها.

إذا فهمت المنطق،
فلن تحتاج إلى الحفظ.

# أولًا: لماذا توجد عدة مديري حزم؟

السؤال الطبيعي الذي يطرحه كل مبتدئ هو:

## لماذا لا يوجد مدير حزم واحد فقط لكل توزيعات لينكس؟

الجواب يشبه تمامًا سبب وجود لغات متعددة بين البشر.

ليست المشكلة في عدم الاتفاق،
بل في اختلاف البيئات التي نشأت فيها هذه الأدوات.

كل مدير حزم وُلد داخل مجتمع مختلف،
بحاجات مختلفة،
وفلسفة مختلفة،
وطريقة مختلفة في التفكير حول النظام.

ولهذا ظهرت أدوات متعددة، لا أداة واحدة.

---

## الفلسفات الأربع الكبرى

### APT — في Debian و Ubuntu

نشأ في بيئة تقدّس:

* الاستقرار
* الوضوح
* التدرّج
* تجنّب المفاجآت

فلسفته تقول:

الأفضل أن يكون الأمر واضحًا وآمنًا،
لا سريعًا وغامضًا.

ولهذا يستخدم كلمات كاملة مثل:

`apt install`
`apt remove`
`apt update`

بدل الحروف المختصرة.

---

### DNF — في Fedora و RHEL

تطوّر من عالم الشركات والخوادم.

هنا الأولوية ليست فقط للبساطة،
بل أيضًا لـ:

* الاعتمادية
* التتبع
* إدارة الأخطاء
* القدرة على التراجع

ولهذا يمتلك ميزات مثل:

`dnf history`

التي تسمح لك بمراجعة العمليات السابقة والتراجع عنها.

فلسفته تقول:

لا يكفي أن تعمل…
يجب أن تعرف ماذا حدث، وكيف تعود للخلف.

---

### Zypper — في openSUSE

بُني بعقلية دقيقة جدًا.

أقرب إلى عقلية المحاسب أو المهندس الصناعي.

التركيز هنا على:

* المعاملات الدقيقة
* إدارة المستودعات
* الفصل بين أنواع التحديث
* الأمان الإداري

ولهذا يفرّق بوضوح بين:

`zypper patch`
`zypper update`
`zypper dup`

فلسفته تقول:

ليس كل تحديث هو نفس الشيء.

---

### Pacman — في Arch Linux

وُلد من فلسفة مختلفة تمامًا:

* البساطة الصارمة
* الشفافية الكاملة
* لا سحر مخفي
* المستخدم مسؤول

لا يحاول Pacman حمايتك من نفسك،
بل يعطيك الأدوات مباشرة.

ولهذا يستخدم:

`pacman -S`
`pacman -Q`
`pacman -R`

بدل الجمل الطويلة.

فلسفته تقول:

أنا لا أشرح كثيرًا،
لأنني أفترض أنك تريد الفهم، لا الحفظ.

---

## الخلاصة

اختلاف الأوامر ليس فوضى.

وليس دليلًا على أن لينكس معقّد بلا سبب.

بل هو انعكاس مباشر لاختلاف الفلسفات التي بُنيت عليها التوزيعات.

APT لا يشبه Pacman
لأن Ubuntu لا تفكر مثل Arch.

وDNF لا يشبه Zypper
لأن Fedora لا تُدار مثل openSUSE.

عندما تفهم هذا،
تتوقف عن سؤال:

## ما هو الأمر الصحيح؟

وتبدأ بسؤال أفضل:

## كيف يفكر هذا النظام؟


## هناك دائمًا عالمان

### العالم الخارجي

مكان الحزم المتاحة للتنزيل.

### العالم الداخلي

ما هو موجود فعلًا على جهازك.

---

# السؤال الذهبي قبل أي أمر

اسأل نفسك دائمًا:

## هل أنا أتعامل مع الخارج أم الداخل؟

هل أبحث في المستودعات؟
أم أبحث فيما هو مثبّت عندي؟

هذه الفكرة وحدها تكفي لفهم أغلب مديري الحزم.

# 1. Pacman — الجرّاح

## الفلسفة

أنا أفترض أنك تعرف ما تفعل.

لا أشرح كثيرًا،
لا أجمّل،
لا أخفي.

كل شيء مباشر وصريح.

---

## المنطق

يعتمد على الحروف لا الكلمات.

``` 
pacman -[ACTION][modifiers]
```

---

## الأفعال الأساسية

* `-S` = Sync
  التعامل مع الخارج (المستودعات)

* `-Q` = Query
  التعامل مع الداخل (المثبت محليًا)

* `-R` = Remove
  الإزالة

* `-U` = Upgrade / Install local file
  تثبيت ملف محلي

---

## قاعدة اليد اليمنى واليسرى

* اليد اليمنى تمتد للخارج
  → `-S`

* اليد اليسرى تبقى قريبة من الجسد
  → `-Q`

طريقة بسيطة جدًا للحفظ.

---

## أمثلة مهمة

### تحديث النظام

``` 
sudo pacman -Syu
```

المعنى:

* Sync
* Refresh lists
* Upgrade everything

ولهذا يسمّيه البعض:

## أمر النجاة

---

### البحث في المستودعات

``` 
pacman -Ss firefox
```

---

### البحث في المثبت محليًا

``` 
pacman -Qs firefox
```

---

### التثبيت

``` 
sudo pacman -S firefox
```

---

### تثبيت ملف محلي

``` 
sudo pacman -U file.pkg.tar.zst
```

---

### الحذف الكامل

``` 
sudo pacman -Rns firefox
```

---

# 2. APT — أمين المكتبة المحافظ

## الفلسفة

الوضوح أهم من الاختصار.

نستخدم كلمات كاملة
لا رموزًا غامضة.

---

## `apt` و `apt-get`

هناك أداتان مهمتان:

### `apt`

النسخة الحديثة للبشر.

* أوضح
* شريط تقدم ملون
* مخرجات مختصرة

---

### `apt-get`

النسخة التقليدية للسكربتات.

* أكثر استقرارًا
* مناسبة للأتمتة
* أفضل في Shell Scripts

---

## ملاحظة احترافية

في السكربتات استخدم:

``` 
apt-get
```

وليس:

``` 
apt
```

لأن `apt` يعرض التحذير:

```text
WARNING: apt does not have a stable CLI interface
```

---

## الرقصة الثنائية الشهيرة

### أولًا

``` 
sudo apt update
```

هذا لا يثبّت شيئًا.

فقط يحدّث فهرس المكتبة.

---

### ثانيًا

``` 
sudo apt upgrade
```

هنا يبدأ التحديث الفعلي.

---

## لماذا خطوتان؟

لأن Debian تقدّس الاستقرار.

أولًا تنظر إلى القائمة الجديدة،
ثم تقرر التنفيذ.

---

## أمثلة مهمة

### التثبيت

``` 
sudo apt install firefox
```

### الحذف

``` 
sudo apt remove firefox
```

### الحذف الكامل

``` 
sudo apt purge firefox
```

### إزالة الحزم اليتيمة

``` 
sudo apt autoremove
```

### البحث

``` 
apt search firefox
```

# 3. DNF — المصلح الحديث

## الفلسفة

YUM كان جيدًا…

لكنه أصبح قديمًا.

أنا النسخة الأذكى والأسرع.

---

## الفرق بين `update` و `upgrade`

عمليًا:

``` 
dnf update
```

و

``` 
dnf upgrade
```

متشابهان جدًا.

وذلك للحفاظ على التوافق مع YUM القديم.

لكن الأفضل استخدام:

``` 
dnf upgrade
```

---

## تحديث مع Refresh

``` 
sudo dnf upgrade --refresh
```

هذا يجمع:

* check-update
* upgrade

في خطوة واحدة.

---

## السلاح السري

``` 
dnf history
```

يمكنك رؤية كل العمليات السابقة.

بل وحتى التراجع عنها:

``` 
sudo dnf history undo last
```

وكأنك تملك Git للحزم.

---

## أمثلة مهمة

### التثبيت

``` 
sudo dnf install firefox
```

### البحث

``` 
dnf search firefox
```

### معرفة من يوفّر ملفًا

``` 
dnf provides /usr/bin/firefox
```

ميزة ممتازة جدًا.

---

# 4. Zypper — المحاسب الدقيق

## الفلسفة

كل شيء يجب أن يكون محسوبًا
وقابلًا للتراجع.

التحكم أهم من السرعة.

---

## ثلاث درجات للتحديث

### تحديثات أمنية فقط

``` 
sudo zypper patch
```

---

### تحديث يومي عادي

``` 
sudo zypper update
```

أو

``` 
sudo zypper up
```

---

### ترقية التوزيعة كاملة

``` 
sudo zypper dup
```

اختصار:

```text
Distribution Upgrade
```

ويُستخدم عادة عند الانتقال بين إصدارات openSUSE.

أما `update` فهو للتحديثات اليومية.

---

## أمثلة مهمة

### تحديث المستودعات

``` 
sudo zypper refresh
```

أو

``` 
sudo zypper ref
```

---

### التثبيت

``` 
sudo zypper in firefox
```

---

### الحذف

``` 
sudo zypper rm firefox
```

# رابعًا: حجر رشيد

## نفس المهمة بأربع لغات

| المهمة        | Pacman      | APT           | DNF            | Zypper    |
| ------------- | ----------- | ------------- | -------------- | --------- |
| تحديث الفهرس  | `-Sy`       | `update`      | `check-update` | `refresh` |
| ترقية النظام  | `-Syu`      | `upgrade`     | `upgrade`      | `update`  |
| التثبيت       | `-S pkg`    | `install pkg` | `install pkg`  | `in pkg`  |
| الحذف         | `-R pkg`    | `remove pkg`  | `remove pkg`   | `rm pkg`  |
| حذف كامل      | `-Rns`      | `purge`       | `remove`       | `rm -u`   |
| البحث         | `-Ss`       | `search`      | `search`       | `se`      |
| عرض المعلومات | `-Si / -Qi` | `show`        | `info`         | `info`    |

---

## البحث بالاسم فقط vs الوصف

| المدير | البحث بالاسم فقط                  | البحث في الوصف       |
| ------ | --------------------------------- | -------------------- |
| Pacman | `pacman -Ss ^firefox`             | `pacman -Ss firefox` |
| APT    | `apt search --names-only firefox` | `apt search firefox` |
| DNF    | `dnf list firefox*`               | `dnf search firefox` |
| Zypper | `zypper se -n firefox`            | `zypper se firefox`  |

# الخطأ الثاني

## التحديث الجزئي في Arch

هذا خطير جدًا:

``` 
sudo pacman -Sy firefox
```

لماذا؟

لأن قاعدة البيانات المحلية تصبح محدثة،
لكن بقية النظام لا يزال قديمًا.

مثال:

إذا تم تحديث مكتبة مثل:

```text
glibc
```

في المستودعات،
فقد تحاول البرامج القديمة العمل مع مكتبة جديدة،
وهنا تبدأ مشاكل التبعيات والانكسار.

الصحيح:

``` 
sudo pacman -Syu
```

أو:

``` 
sudo pacman -Syu firefox
```

# سابعًا: أسئلة شائعة

---

## هل يجب استخدام sudo دائمًا؟

نعم،
لأي أمر يغيّر النظام:

* تثبيت
* حذف
* تحديث

ولا تحتاجه غالبًا في:

* البحث
* الاستعلام
* القراءة

---

## ما الفرق بين remove و purge؟

### remove

حذف البرنامج
مع الاحتفاظ بالإعدادات.

---

### purge

حذف البرنامج
وحذف الإعدادات بالكامل.

---

## ما الذي لا يفعله مدير الحزم؟

مثال:

``` 
sudo apt install nginx
```

هذا يثبّت البرنامج فقط.

لكن:

``` 
sudo systemctl start nginx
```

هذا يشغّله.

مدير الحزم مسؤول عن:

* التثبيت
* التحديث
* الحذف

وليس مسؤولًا عن:

* تشغيل الخدمات
* إدارة المستخدمين
* إدارة الملفات

عاشرًا: Shell الأذكى - Fish vs Zsh

الـ shell التي تستخدمها تؤثر على تجربتك مع مديري الحزم.
المشكلة مع    العادي

  

# تكتب:
sudo pacman -S fire<TAB>

# تحصل على:
fire  firebird  firefox  firewalld...
# (مجرد قائمة بلا شرح)

الحل: Fish أو Zsh

  

# تكتب:
sudo pacman -S fire<TAB>

# تحصل على (مع أوصاف):
firefox          -- متصفح ويب
firewalld        -- برنامج جدار ناري
firejail         -- أداة عزل

Fish Shell — الأفضل للمبتدئين
التثبيت

  

# Ubuntu/Debian
sudo apt install fish

# Arch
sudo pacman -S fish

# Fedora
sudo dnf install fish

# openSUSE
sudo zypper in fish

# جعلها الافتراضية
chsh -s /usr/bin/fish

لماذا Fish أفضل للمبتدئين؟
1. اقتراحات تلقائية من السجل

  

# سبق أن كتبت:
sudo pacman -Syu

# المرة القادمة، اكتب فقط:
sudo pac
# Fish تقترح (بخط رمادي): sudo pacman -Syu
# اضغط → للقبول

2. تكملة من صفحات man تلقائيًا

  

# اكتب:
pacman -<TAB>

# تحصل على كل الخيارات مع الشرح:
-S  -- تثبيت حزمة
-Q  -- الاستعلام عن قاعدة البيانات
-R  -- حذف حزمة

3. تكملة أسماء الحزم

  

# يعمل مباشرة:
sudo pacman -S <TAB>
# يعرض الحزم المتاحة

sudo pacman -R <TAB>
# يعرض المثبتة فقط

4. تلوين الأوامر

  

# أمر صحيح = أخضر
sudo pacman -S firefox  # أخضر

# علَم خاطئ = أحمر
sudo pacman -Z firefox  # أحمر (Z غير موجود)

5. تعمل من الصندوق

لا تحتاج إعداد معقد.
ثبّت واستخدم مباشرة.
Zsh — الأقوى للمتقدمين
التثبيت مع Oh-My-Zsh

  

# تثبيت zsh
sudo apt install zsh  # أو dnf, pacman, zypper

# تثبيت Oh-My-Zsh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

# جعلها الافتراضية
chsh -s /usr/bin/zsh

الإضافات المفيدة

عدّل ~/.zshrc:

  

plugins=(
    git
    ubuntu          # إن كنت على Ubuntu (اختصارات apt)
    archlinux       # إن كنت على Arch (اختصارات pacman)
    dnf             # إن كنت على Fedora
    sudo            # اضغط ESC مرتين لإضافة sudo
    command-not-found  # اقتراح الحزمة للتثبيت
)

لماذا Zsh قوية؟
1. اختصارات مدمجة حسب التوزيعة

إضافة Arch:

  

pacin   = sudo pacman -S      # تثبيت
pacr    = sudo pacman -R      # حذف
pacsyu  = sudo pacman -Syu    # تحديث
pacss   = pacman -Ss          # بحث

إضافة Ubuntu:

  

agi     = sudo apt install
agr     = sudo apt remove
agu     = sudo apt update && sudo apt upgrade
acs     = apt search

2. command-not-found

  

# تكتب:
htop

# Zsh تجيب:
الأمر 'htop' غير موجود، لكن يمكن تثبيته بـ:
sudo apt install htop

3. إضافة sudo (السحر)

  

# كتبت:
pacman -Syu
# اضغط Enter → permission denied

# اضغط ESC ESC (مرتين بسرعة)
# الأمر يصبح:
sudo pacman -Syu
